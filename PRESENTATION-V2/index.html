Part 2 · Dine‑In App

Scan, Order, Relax

Customers order from their phone — Scan a QR code on the table or open the Telegram mini app. Browse the menu, add items, enter your daily access code, and submit. The kitchen gets the order instantly. Receipt arrives via Telegram. No staff needed to take orders.
📱
Step 1: Browse Menu
Collapsible categories with search. Each item shows emoji, name, description, and dine-in price. Add to cart with quantity controls.
🛒
Step 2: Review Cart
Items with quantity increment/decrement, line totals, grand total. Choose order type: Dine-In or Takeaway.
🔑
Step 3: Enter Access Code
For dine-in: enter daily 4-6 digit code. SHA-256 hash verified server-side via Supabase RPC. Prevents fraudulent remote orders.
✅
Step 4: Order Confirmed
Order inserted to Supabase. Receipt PNG generated via html2canvas. Sent to customer's Telegram. Staff group notified with order details and "Prepare Order" button.
🌙
Dark & Light Mode
Auto-adapts to phone's theme. Toggle anytime. Dynamic theme-color meta tag.
Part 2 · Walkthrough

Ordering in 5 Simple Steps

From scanning the QR code to the kitchen receiving the order — the entire flow in detail.

📱
Step 1
Open App
Scan table QR code or open Telegram mini app from /dinein command. Telegram.WebApp SDK expands the app fullscreen.
🍽️
Step 2
Browse & Add
Collapsible categories with search. Items show emoji, name, description, price. Tap + to add. Sticky cart footer shows running total.
🛒
Step 3
Checkout
Review cart with qty controls. Choose Dine-In or Takeaway. Add optional note. For dine-in: enter access code.
🔑
Step 4
Verify Code
Full-screen modal for daily access code entry. Supabase RPC verifies SHA-256 hash. Error/success states. Ensures physical presence.
✅
Step 5
Order Sent
Thank you screen with table number. Receipt PNG sent to Telegram. Staff group notified with "Prepare Order" inline button.
⚡
Total Time: Under 30 Seconds
From scanning the QR code to the kitchen receiving the order. Receipt in Telegram. Staff notified. No human intervention needed.
Part 3 · Delivery App

Order Delivery,
Track It Live

Customers order delivery through a dedicated mini app. Browse the menu with delivery-specific pricing, share GPS location, choose delivery or pickup, and receive real-time updates via Telegram. Riders claim orders through Telegram group chat with inline action buttons.
🍽️
Delivery Menu
Same items as dine-in but with separate delivery pricing (price_delivery column). Collapsible categories with search.
📍
GPS Location
Browser geolocation with high accuracy option. Manual address input as fallback. Driver uses pin to navigate.
🔄
Delivery or Pickup
Toggle in checkout. Delivery requires address + phone. Pickup requires name + phone. Both types submit to same orders table.
📷
Receipt Generation
Styled "DELIVERY RECEIPT" PNG via html2canvas sent to customer's Telegram chat. Shows items, total, delivery address, phone.
🤖
Driver Dispatch Webhook
Pickup orders sent to driver group immediately. Delivery orders wait for admin confirmation. "Take Order" button → driver assigned → "Mark Delivered" completes cycle. Live location forwarding.

Part 4 · Admin App

The Command Center

13 integrated tabs for complete restaurant control. Dashboard analytics, Kanban order management, menu CRUD, inventory tracking, sales reports, customer intelligence, rider dispatch, table reservations, access code security, user permissions, profile management, and global settings.
📊
Dashboard
5 KPI cards: total orders, delivery revenue, dine-in revenue, avg order, active orders. 7-day revenue bar chart (delivery vs dine-in). Recent orders list, popular items ranking, top riders leaderboard, staff-on-shift status, pending quick-approve actions. "Since last login" summary.
📋
Orders Kanban Board
Horizontal scrollable lanes: Pending → Confirmed → Delivering/Dining → Completed/Cancelled. Each card shows customer name, order type badge, time, phone, address, total, rider. Click for detail panel with Telegram chat link, embedded map, rider assignment. Filter by order type and date. Compact mode toggle.
📈
Reports & Analytics
Date range picker. Summary cards, delivery vs pickup breakdown, hourly/daily/weekly trends. Delivery location heatmap. Popular items ranking. Cancelled order analysis. CSV and PDF export. Send report to Telegram.
Part 4 · Management

Menu, Stock & Customers

Complete control over your products and customer relationships. Manage menu items with separate dine-in/delivery pricing, track inventory with auto-deduction, and understand your customers with spending tiers and order history.
📂 Menu Control
Create/edit/delete items and categories. Separate dine-in (price_pickup) and delivery (price_delivery) pricing per item. Upload photos or emoji icons. Toggle availability on/off. Bulk show/hide/delete. Filter by low stock, no image, no category. Display order control.
📦 Inventory
Full item list with stock quantities, low stock thresholds, cost prices. Auto-deduction mode (on order placement) or manual mode. Filters: all/low stock/out of stock. Bulk adjustments (restock/delivery/waste/correction). Stock log history. 7-day usage forecast. Category-based filtering.
👥
Customers
Aggregated customer list from order data. Shows total orders, total spent, AOV, last order date. Tiers: VIP/Regular/New. Click for order history, favorite items, private notes. Telegram profile photo, username, direct chat link. Sort by spent/orders/recent. Filter by tier/period. Export CSV.
🔐
Dine-In Security
Generate daily 4-6 digit access codes. Save to database. Send to Telegram group. Used by dine-in customers to verify physical presence before ordering. SHA-256 hash verified via Supabase RPC.
Part 4 · Operations

Staff, Riders & Settings

Manage your team and configure every aspect of your restaurant. Role-based admin access, delivery rider profiles with performance tracking, table reservations, and 6-panel global settings for complete control.
👤
User Management
Create/edit/delete admin users. Roles: admin (full access), manager (no riders/users), order_man (orders/dashboard/customers only), user (dashboard+profile). Granular permission toggles per tab. Telegram account linking. Cash shift management (clock in/out with amounts and notes).
🛵
Rider Management
CRUD for delivery drivers: name, phone, commission rate, Telegram user ID. Per-rider stats: claimed, active, completed, cancelled orders, total revenue, avg delivery time. Active/inactive toggle. Telegram group integration for order dispatch.
📅
Reservations
Date-picker view for table bookings. Create/edit/delete with name, phone, party size, time, notes. Status flow: pending → confirmed → seated → completed/cancelled. Push notification on new reservation to Telegram.
⚙️ 6-Panel Global Settings
🏪 General
Restaurant name, phone, address, timezone, operating hours
💰 Pricing & Tax
Currency, tax name/rate, service charge, min order amount
🚚 Delivery
Delivery fee, free threshold, min order, auto-confirm toggle, payment methods (Cash, Card, QR)
🔔 Push Notifications
Telegram bot tokens (pickup, delivery, reports), group IDs, webhook URLs
🧾 Receipt & Display
Receipt header/footer text, low stock threshold, items per page, dark/light theme
⚙️ Advanced
Order prefix (ORD-), localization, low stock threshold (default: 5)
